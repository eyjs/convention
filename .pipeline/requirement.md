# 요구사항: 관리자 페이지 모바일 대응

## 개요
관리자 페이지 전체를 모바일에서 사용 가능하도록 개선한다.
현장에서 모바일로 즉석 대응하는 케이스가 빈번함 (일정변경, 알림발송, 호텔/룸번호 수정 등).
PC가 주 환경이지만, 엑셀 업로드를 제외한 모든 기능이 모바일에서도 동작해야 한다.

## 사용 시나리오
- 현장에서 호텔 체크인 후 룸번호 10건 정도 모바일로 즉석 수정
- 일정 변경사항 현장에서 즉시 반영
- 긴급 알림/SMS 현장에서 즉시 발송
- 참석자 정보 현장에서 빠르게 검색/확인

## 공통 원칙
- 테이블 → 모바일에서 **카드 뷰**로 전환, 상세는 **모달**
- 가로 나열 필터/버튼 → 모바일에서 **세로 스택** 또는 **flex-wrap**
- 검색 입력 → 모바일에서 **풀 너비** 확보
- 고정 폭(`w-20`, `w-28`, `w-64` 등) → **반응형 폭**으로 교체
- `whitespace-nowrap` → 모바일에서 **해제**
- 기존 PC 레이아웃은 **변경하지 않음** (md: 이상에서 기존 유지)

## 대상 화면 및 수정 사항 (13개)

### CRITICAL (사용 불가)

#### 1. 참석자관리 (GuestManagement.vue)
- **문제**: 검색+필터+버튼이 한 줄 `flex`에 있어 모바일에서 검색창 1글자만 보임
- **수정**: 
  - 검색 입력: `w-full` (모바일에서 한 줄 독립)
  - 필터(셀렉트+버튼): 아래 줄로 분리 (`flex-col sm:flex-row`)
  - 테이블: `hidden md:block` 유지 + 모바일 카드 뷰 보강 (현재 카드 뷰 있음 → 검색/필터 연동 확인)

#### 2. 여행배정 (TravelAssignmentManager.vue)
- **문제**: 테이블이 `hidden md:block`으로 모바일에서 완전 숨김, 카드 뷰 데이터 불완전
- **수정**:
  - 모바일 카드 뷰에 모든 배정 필드 표시 (룸번호, 룸메이트, 버스, 테이블 등)
  - 카드에서 직접 편집 가능 (터치해서 모달로 수정)
  - 일괄배정 입력 `w-20`/`w-28` → `w-full sm:w-20` 반응형

### MAJOR (사용 어려움)

#### 3. 일정관리 (ScheduleManagement.vue)
- **문제**: 코스 필터 버튼이 가로 스크롤, 옵션투어 탭 + 버튼도 가로 배치
- **수정**:
  - 필터 버튼: `flex-wrap` 적용 또는 `grid grid-cols-2 sm:flex`
  - 일정 카드/목록: 모바일에서 세로 스택
  - 추가/편집 버튼: 모바일에서 풀 너비 또는 FAB 스타일

#### 4. 액션관리 (ActionManagement.vue)
- **문제**: 카테고리 카드 그리드 작은 화면에서 텍스트 잘림
- **수정**:
  - 그리드: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`
  - 카드 내 텍스트 `truncate` → `line-clamp-2`
  - 설정 폼 모달: 모바일 풀 너비

#### 5. 알림발송 (NotificationSender.vue)
- **문제**: 6개 타입 필터 2열 그리드가 좁은 화면에서 찌그러짐, 모바일에서 기능 동작 안 함
- **수정**:
  - 타입 필터: `grid-cols-2 sm:grid-cols-3 lg:grid-cols-6`
  - **앱 환경(Capacitor WebView)에서 페이지 진입 불가 원인 조사 및 수정** (웹에서는 정상 동작, 앱에서만 터치 시 무반응 — JS 에러/권한/WebView 호환성 확인)
  - 발송 폼: 모바일에서 세로 스택

#### 6. SMS 발송 (SmsManagement.vue)
- **문제**: 수신자 목록 `max-h-[200px]`로 20명 이상 시 스크롤 불가, 발송이력 테이블 PC 기준
- **수정**:
  - 수신자 목록: `max-h-[50vh]` 반응형
  - 에디터+수신자 2열 → 모바일에서 세로 스택 (현재 `lg:grid-cols-2` → 유지)
  - 발송이력: 모바일 카드 뷰 추가

#### 7. 게시판관리 (BoardManagement.vue)
- **문제**: `whitespace-nowrap`으로 제목 잘림, 날짜/조회수 컬럼 좁음
- **수정**:
  - 모바일: 카드 뷰 전환 (제목 + 카테고리 + 날짜)
  - PC: 기존 테이블 유지
  - `whitespace-nowrap` → `whitespace-normal md:whitespace-nowrap`

### MINOR (사소한 UX)

#### 8. 여행배정 일괄 컨트롤 (TravelAssignmentManager.vue)
- **문제**: 일괄배정 입력 `w-20`/`w-28` 고정폭, `gap-y` 없어 줄바뀜 시 간격 없음
- **수정**: `w-full sm:w-20`, `gap-2` → `gap-x-2 gap-y-2`

#### 9. 옵션투어 (OptionTourSurveyManagement.vue)
- **문제**: 테이블에 모바일 뷰 없음
- **수정**: 모바일 카드 뷰 추가 (투어명 + 날짜 + 참석자수)

#### 10. 폼빌더 (FormBuilderEditor.vue)
- **문제**: `gap-6`이 모바일에서 과도
- **수정**: `gap-3 lg:gap-6`

#### 11. 대시보드 (DashboardOverview.vue)
- **문제**: `grid-cols-2`가 320px에서 좁음
- **수정**: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-5`

#### 12. 인쇄미리보기 (TablePrint.vue)
- **문제**: 사이드바 `w-96` 고정
- **수정**: `w-full md:w-96`

## 제외 사항
- 엑셀 업로드 관련 기능은 모바일 대응 제외 (PC 전용 유지)
- 좌석배치도 에디터 (SeatingLayoutEditor)는 PC 전용 유지

## 기술 제약
- 프론트엔드: Vue 3 + JavaScript (TypeScript 아님)
- CSS: Tailwind CSS
- 기존 PC 레이아웃 변경 금지 — `sm:`/`md:`/`lg:` 브레이크포인트로 모바일만 추가
- 새 컴포넌트 최소화 — 기존 컴포넌트에 반응형 클래스 추가 우선

## 우선순위
1. 참석자관리 검색 + 여행배정 카드 뷰 (CRITICAL)
2. 일정관리 + 알림발송 동작 불가 수정 (MAJOR, 현장 필수)
3. SMS 발송 + 게시판관리 (MAJOR)
4. 액션관리 + 나머지 MINOR 일괄
