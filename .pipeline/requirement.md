# 요구사항: 관리자 페이지 전체 모바일 대응

## 현재 상태

| 페이지 | 줄 수 | 테이블 | 반응형 | 모바일 대응 |
|---|---|---|---|---|
| 참석자관리 | 450+ | ✅ 카드+테이블 이중 | ✅ 완료 | ✅ |
| 게시판관리 | 788 | 32개 th/td | 패딩만 | ❌ 테이블 가로스크롤 |
| 일정관리 | 1025 | 없음 (카드) | 일부 | ⚠️ 버튼/카드 조정 필요 |
| 액션관리 | 1650 | 없음 (카드) | 없음 | ❌ 전체 재정비 |
| 설문관리 | 372 | 8개 th/td | 1개 | ❌ 테이블 모바일 |
| 폼빌더 | 287 | 없음 | 없음 | ⚠️ 카드라 기본 OK |
| 여행배정 | 546 | 15개 th/td | 없음 | ❌ 테이블 모바일 |
| 좌석배치도 목록 | 200 | 없음 (카드) | 일부 | ⚠️ 기본 OK |

## 작업 대상 (우선순위)

### Tier 1 — 현장에서 자주 쓰는 것
1. **게시판관리** — 공지 목록 테이블 → 모바일 카드
2. **일정관리** — 이미 카드 기반이라 세부 조정만
3. **설문관리** — 설문 목록 테이블 → 모바일 카드

### Tier 2 — 가끔 쓰는 것
4. **액션관리** — 카드 기반이라 패딩/간격 조정
5. **여행배정** — 테이블 → 모바일 카드

### Tier 3 — 사전 준비용 (PC에서 주로)
6. **폼빌더** — 이미 카드 기반, 큰 문제 없음
7. **좌석배치도 목록** — 카드 기반, OK

## 공통 패턴 (참석자관리와 동일)
```
<!-- 모바일 카드 -->
<div class="md:hidden space-y-2">
  <div v-for="item" class="bg-white rounded-lg shadow-sm p-3 flex items-center gap-3">
    핵심 정보만 (제목 + 상태 + 날짜)
  </div>
</div>

<!-- PC 테이블 -->
<div class="hidden md:block">
  <table>기존 테이블</table>
</div>
```
