# 남은 과제 작업계획

## 1. 좌석배치도 저장 안정화 (CRITICAL)
### 현상
- 핀 배치 후 사용자 페이지 이동 → 데이터 유실 간헐 발생
- 자동 저장(3초 디바운스) 전에 페이지 이탈 시 유실

### 원인
- `toLayoutJSON()`이 캔버스 상태 불안정 시 빈 배열 반환
- `onUnmounted`의 `saveNow()`가 비동기라 완료 전 컴포넌트 소멸

### 해결안
1. **저장 버튼 클릭 시 즉시 저장 + "저장됨" 피드백** (현재 구현됨 — 검증 필요)
2. **`beforeRouteLeave` 가드**: 미저장 변경 있으면 "저장하지 않고 이탈?" 확인
3. **`navigator.sendBeacon`**: 페이지 이탈 시 비동기 저장 보장
4. **빈 데이터 저장 방지 강화**: `pins.length === 0`이면 절대 PUT 안 함 + 서버에서도 검증

## 2. 사용자 뷰 (MySeatView) 안정화
### 현상
- +/- 줌 후 빈 영역
- 맞춤 버튼 동작 불안정
- 핀치줌 미지원 (브라우저 네이티브 의존)

### 해결안
1. **CSS `touch-action: pinch-zoom`** 적용 → 이미지 영역에서 네이티브 핀치줌 활성화
2. **줌 시 컨테이너 크기 = 이미지 크기 × 줌** 정확히 계산 → 빈 영역 제거
3. **맞춤 버튼**: `fitZoom` 값으로 복원 + 스크롤 (0,0) 리셋
4. **핀 크기 줌 비례** 유지 (현재 구현됨)

## 3. 알림톡 딥링크 실제 테스트
### 할 일
1. Popbill 템플릿 "좌석안내" (`026040000793`) 버튼 URL에 `#{버튼1}` 설정 확인
2. 테스트 발송 → 카카오톡 수신 → 버튼 클릭 → auto-login → 내 자리 확인
3. AccessToken이 없는 사용자 (일반 User Role) 대응: 로그인 페이지로 fallback

## 4. 관리자 모바일 UI 세부 조정
### 할 일
- **참석자관리**: 테이블 → 모바일에서 카드 리스트로 전환 (`hidden md:table-cell` + 모바일 카드)
- **게시판관리**: 동일 패턴
- **SNS발송**: 수신자 체크박스 목록 높이 조절, 변수 버튼 그리드화
- **대시보드**: 통계 카드 폰트 크기 반응형
